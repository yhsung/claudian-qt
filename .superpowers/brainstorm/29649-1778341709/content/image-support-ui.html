Section 1: UI behavior

Proposed behavior

  • Users can add multiple images by file picker, drag/drop, or clipboard paste.
  • Each pending attachment shows a thumbnail, filename, and remove action before send.
  • Reopened user messages display a thumbnail gallery above the message text.
  • Clicking a history thumbnail opens a larger preview modal without leaving the chat.

Intentional limits

  • v1 supports image files only, not PDFs or arbitrary documents.
  • Attachments are sent only with the current user turn; no retroactive edits.
  • Thumbnail rendering uses managed copies stored by the app, not original file paths.
